# VibeCheck 产品规划书 (Product Roadmap)

> **Slogan**: 懂你的情绪，更懂你的歌 (Understanding your vibe, discovering your song)

## 1. 产品愿景 (Vision)
打破传统的“基于听歌历史”的推荐茧房，打造一款**真正基于“情绪与内容”的深度音乐搜索引擎**。让用户在无法用歌名准确表达需求时，能通过通过描述心情、场景或故事，找到直击心灵的音乐。

## 2. 解决的用户痛点 (User Pain Points) - *Deep Dive*

| 痛点场景 | 深度洞察 (Deep Insight) | 现有方案缺陷 | VibeCheck 差异化价值 |
| :--- | :--- | :--- | :--- |
| **"只可意会"的模糊搜索** | 用户常有"通感"需求，用视觉/嗅觉/故事描述听觉。例："找一首像王家卫电影里那样湿漉漉的歌"。 | 关键词匹配失效。搜"湿漉漉"出来的歌名可能带"雨"，但不代表意境是那样。 | **语义映射 (Semantic Mapping)**: 利用向量库，将"湿漉漉"这种通感词汇，映射到歌词中描述的阴雨、潮湿、胶片感意境。 |
| **信息茧房与审美疲劳** | 推荐算法本质是"猜你喜欢"，为了提高点击率，往往只推你习惯的曲风，导致用户听歌越来越窄。 | 协同过滤(CF)算法倾向于推"热门"和"相似"，牺牲了"惊喜感"和"长尾内容"。 | **内容挖掘 (Content Discovery)**: 抛弃热度权重。完全基于歌词内容的相似性推荐。帮你找到那首**冷门但灵魂契合**的宝藏歌 (Deep Cuts)。 |
| **场景歌单的动态需求** | 用户需要的不是静态列表，而是"动态陪伴"。例："我要写代码，先来点轻音乐进入状态，半小时后要提神"。 | 静态歌单无法随心而动，且千篇一律。 | **生成式歌单 (Generative Playlist)**: AI 理解你的 Storyline，生成有起承转合的歌曲序列。 |
| **情感共鸣的"不可检索性"** | **(用户反馈重点)** 网易云评论区确实有海量情感故事，但它们是**散落与不可检索**的。你必须先点进歌，才能看到评论。 | 如果我此刻心情是"像失恋了一样难过"，我无法反向通过这个心情去检索那些"评论区都在哭"的歌。 | **情感索引 (Emotional Indexing)**: AI 的价值不在于"写书评"，而在于**做索引**。AI 将歌词的情感浓度量化并索引，让用户能**通过情绪反向定位歌曲**。 |

## 3. 面向毕设的核心亮点 (Project Highlights)
1.  **AI Native 搜索引擎**: 将传统的 SQL 如果查询升级为 Vector 语义检索。
2.  **数据资产化**: 展示如何将 10万+ 非结构化的文本数据（歌词），通过 LLM/Embedding 转化为结构化的数据资产（向量索引）。
3.  **情绪量化算法**: 不仅仅是 RAG，还展示了如何用 AI 对抽象的"Vibe"进行数学建模。
4.  **深度用户画像 (Deep Profiling)**: 超越简单的"听歌记录"，通过用户搜索的自然语言（如总是搜"下雨"、"孤独"），构建可视化的**心理/情绪画像**。

## 4. 功能规划 (Feature Roadmap)

### Phase 1: 核心基础 (MVP) - *已完成爬虫部分*
- [x] **全量数据构建**: 10万首热门华语歌曲、歌词。
- [ ] **语义检索引擎 (The Core)**:
    - 核心能力："说人话"搜歌。支持自然语言描述场景、心情、故事。
    - 技术栈：SiliconFlow BGE-M3 Embedding + PostgreSQL `pgvector`。
- [ ] **"情感雷达" (Vibe Radar)**:
    - *替代原 AI 乐评*。不生成废话文本，而是生成该歌曲的**情感维度图** (如：悲伤值 80%，治愈值 20%，怀旧值 60%)，直观展示歌曲基调。

### Phase 2: 进阶功能 (毕设加分项) - *深度个性化*
- [ ] **"灵魂画像" (Soul Persona)**:
    - 根据用户的搜索历史（Story/Vibe），生成可视化的 AI 画像。
    - 例："你是'深夜哲学家'，喜欢在凌晨2点搜索关于'遗憾'的歌。"
- [ ] **个性化语义重排序 (Personalized Rerank)**:
    - 同样的搜索词 "治愈"，对于喜欢摇滚的人推荐的是 "Linkin Park"，对于喜欢民谣的人推荐的是 "陈粒"。
- [ ] **"听觉通感" (Synesthesia Search)**:
    - 允许搜索"颜色"（灰色的歌）、"温度"（冰冷的歌）、"触感"（粗糙的歌）。

### Phase 3: 展望 (未来迭代) - *多模态扩展 (Multimodal)*
- [ ] **"以图搜歌" (Image-to-Vibe)**:
    - **原理 (User Insight)**: 正如您所见，本质是 **Visual-to-Text**。
    - **流程**: 用户上传一张"夕阳下的海边"照片 -> 多模态大模型 (如 GPT-4V / CLIP) 识别场景与意境，生成描述词 "Sunset, loneliness, gentle waves, nostalgia" -> 投入 VibeCheck 语义搜索引擎 -> 找到歌词描写了同样画面的歌。
- [ ] **反向情感推荐**: "既然你喜欢看网易云红心歌曲里的遗憾故事，这首冷门歌的歌词简直是为你写的"。
- [ ] **Demo 开发**: 简单的 React 前端展示全流程。

## 5. 商业价值潜力 (Business Value) - *Deep Dive*

### B 端：内容创作者的 "歌词美学" (Lyric Aesthetics)
既然我们核心是"懂歌词"，那就服务于需要**通过歌词传达故事**的创作者 (如情感博主、混剪 AMV、旅行 Vlog)。

| 场景 | 痛点 | VibeCheck 商业切入 (核心竞争力：**歌词共鸣**) |
| :--- | :--- | :--- |
| **情感 Vlog / AMV** | "我想剪个'暗恋'主题的视频，需要歌词里隐晦提到'不敢开口'的BGM。" | **歌词意境匹配**: 只有我们能通过语义搜索，精准找到歌词意境完全匹配画面故事的歌曲，而不仅仅是节奏卡点。 |
| **文案配乐** | "写了一段关于'城市孤独'的文案，想找首词也能对上的歌发朋友圈。" | **文案搜歌**: 直接输入文案，系统推荐歌词与文案灵魂契合的配乐。 |
| **广告商用** | "品牌宣传片需要歌词积极向上，绝对不能有负面隐喻。" | **歌词合规检测**: 利用 AI 深度分析歌词含义，确保商用音乐的价值观安全。 |

### C 端：垂直的 "音乐种草" 社区 (The 'Xiaohongshu' of Music)
目前的流媒体 App (网易云/TME) 越来越臃肿，且强于"播放"弱于"发现"。

*   **定位差异**:
    *   **网易云/QQ音乐**: 音乐**消费**平台 (听)。
    *   **VibeCheck**: 音乐**决策**平台 (找)。
*   **商业模式**:
    *   **导流佣金**: 用户在 VibeCheck 种草，点击跳转到 TME/Spotify 播放，赚取 affiliate 佣金。
    *   **小众音乐人宣发**: 独立音乐人买不起大推荐位，但在 VibeCheck 上，只要他的歌词/意境够好，就能通过"长尾搜索"被精准用户搜到。这是**内容为王**的公平流量分发。
